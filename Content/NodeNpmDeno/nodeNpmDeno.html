Node & npm & Deno

1. Node.js

Um die Vorteile von JavaScript wie non-blocking IO und die hohe Nebenläufigkeit durch das Event Driven Design auch serverseitig einsetzen zu können wurde die Google V8 Engine in ein c++ Programm eingebettet. So entstand die Serverseitig einsetzbare Laufzeitumgebung node.js.

Ehemals serverseitig blockierende aufrufe durch phpwie Datenbankabfragen konnten seitdem durch asynchrones JavaScript ersetzt werden.

Viele Asyncrone Methoden unter node.js haben auch ein syncrones Equivalent.

Durch die verwendung von JavaScript auf Client und Server Seite sind JavaScript Entwickler vielseitig einsetzbar. Trotzdem ist node.js nur selten auf aktuellen Webservern vertreten. Dies wird allerdings auch dadurch beinflusst, dass einigen node.js Server hinter Apache Proxys versteckt sind.

Zwischen der Server und Client Seitigen javaScript Entwicklung gibt es einige grundlegende unterschiede.

Browser node.js
Globales Objekt heißt window Globales Objekt heißt global
var x ⇒ window.x var x ⇒ keine Property in global
Keine Module (bis ES6 import) Module mit require() importieren
DOM via document-Objekt Kein document-Objekt, kein DOM (bis auf jsdom)
Sicherheit via Sandbox (kein Zugriff auf lokales Filesystem / Betriebssystem) Ohne Sicherheitsbeschränkungen
Entwicklung für "alle" Browser Entwicklung für genau eine node.js Version

Quelle: vgl. [12.02.2021] https://kaul.inf.h-brs.de/wem/#app-content-7-0&04_dom=page-27&05_async=page-105&06_node=page-18

Originalquelle: https://stackoverflow.com/questions/23959868/differences-between-node-environment-and-browser-javascript-environment

Node.js verwendet eine angepasste Eventloop. Mit setImmediate(callback) kann ein Callback noch vor Ablauf des aktuellen Eventloop Zykluses ausgeführt werden.

Vorsicht vor Livelogs! Rekursive Callbackaufrufe können eine nie leer werdende Microtaskqueue zufolge haben, sodass die Macrotasks der Eventloop nichtmehr abgearbeitet werden können.

Beispiel

Das nachfolgende node.js Kommandozeilen programm, erzeugt eine Textdatei, welche als Kommandozeilen Parameter einen Wert erwartet, welcher die Anzahl der zu erzeugenden Zeilen mit der jeweiligen Zeilennummer und einem "." erwartet. Der Übergabeparameter kann auch einen "_" enthalten, sodass 10_000 = 10000.

2. npm

Npm steht für Node Package Manager und wird verwendet um node.js Pakete zu verwalten. Um die verschiedenen Versionen von node.js selbst zu verwalten kann der nvm (Node Version Manager) verwendet werden. Beide Manager lassen sich nach der Installation via Kommandozeile verwenden.

Geladene Pakete können mittels CommonJS in den JavaScript Namespace geladen werden.
Kurzscreibweise: const package = requiere("packageName"); .
requiere() kapselt eine Reihe komplexerer Prozesse die im Hintergrund durchgeführt werden.

Seit ES6 gibt es auch die Alternative Packete durch
import {functionA, functionB} from "./myFile.mjs"
zu importieren. die zu importierenden Funktionen müssen dabei um das export Schlüsselwort erweitert werden.

Die beiden Methoden sollten allerdings nicht vermischt werden.
Kein requiere in .mjs.
Kein import in .js.
